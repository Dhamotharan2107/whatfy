Authentication

All API endpoints require an active login session. Log in via /auth — the server sets an st session cookie automatically.

How to Authenticate
Session Cookie (Browser)
# Automatically set after login at /auth
Cookie: st=your_session_token
cURL / Scripts — pass the cookie manually
curl https://your-domain.com/api/send \
  -H "Cookie: st=your_session_token" \
  -d '{"number":"919876543210","message":"Hi!"}'
401 response when not authenticated
{"error": "unauthorized"}

Error Codes

All errors return JSON with an error key describing what went wrong.

HTTP StatusMeaningCommon Cause
200SuccessRequest completed successfully
401UnauthorizedNot logged in — session cookie missing or expired
400Bad RequestMissing or invalid request parameters
422UnprocessableRequired field missing in JSON body
502Gateway ErrorWhatsApp Go server is offline or not reachable
500Server ErrorUnexpected internal error

WhatsApp Connection

Manage the WhatsApp session — check status, get the QR code to scan, pair using a phone number, or disconnect.

GET /api/wa/status Check WhatsApp connection state ›

Returns whether WhatsApp is currently connected, logged in, and which phone number is linked. Call this before trying to send messages.

200 OK401 Unauthorized
cURL
curl https://your-domain.com/api/wa/status \
  -H "Cookie: st=TOKEN"
Response
{
  "connected": true,
  "loggedIn": true,
  "phone": "919876543210",
  "name": "My Business"
}
Response Fields
FieldTypeDescription
connectedbooleanTCP connection to Go server is alive
loggedInbooleanWhatsApp session is authenticated
phonestringLinked phone number with country code
namestringWhatsApp display name of the linked account
GET /api/wa/qr-image Get QR code as a base64 PNG image ›

Fetches the current WhatsApp login QR code and returns it as a base64-encoded PNG. Display it in an <img src="..."> tag for the user to scan with WhatsApp → Linked Devices. The QR expires every 20 seconds — refresh regularly until connected.

200 OK400 No QR401 Unauthorized
Success Response
{
  "qr_image": "data:image/png;base64,iVBORw0KGgo..."
}
Already Connected
{
  "error": "No QR available — may already be connected"
}
Usage in HTML
// Poll every 5s until connected
const r = await fetch('/api/wa/qr-image');
const data = await r.json();
if (data.qr_image)
  document.getElementById('qr').src = data.qr_image;
POST /api/wa/pair Pair using phone number instead of QR ›

Request a pairing code for a phone number. The user opens WhatsApp → Settings → Linked Devices → Link with Phone Number and enters the code. Useful on headless servers where displaying a QR image is not practical.

200 OK400 Error401 Unauthorized
Request Body
{
  "number": "919876543210"
}

// Include country code, no + or spaces
Response
{
  "code": "ABCD-1234"
}

// User enters this code in WhatsApp
Body Parameters
FieldTypeRequiredDescription
numberstringRequiredPhone number with country code, no + prefix (e.g. 919876543210)
GET /api/wa/logout Disconnect and clear the WhatsApp session ›

Terminates the active WhatsApp session on the Go server. After calling this, the QR code will reappear and a new scan or pairing is required to reconnect. All running campaigns are not affected but will fail to send until reconnected.

200 OK401 Unauthorized
Response
{
  "status": "disconnected"
}

Messaging

Send text messages, send media files, list conversation history, and stream real-time events.

POST /api/send Send a text message to any WhatsApp number ›

Sends a plain-text WhatsApp message to the specified number. The number must include the country code without the + prefix. WhatsApp must be connected before calling this endpoint.

India numbers: Use 91 prefix — e.g. 919876543210 for +91 98765 43210.
200 OK500 WA Offline401 Unauthorized
Request Body
{
  "number": "919876543210",
  "message": "Your order #42 is confirmed!"
}
Response
{
  "status": "sent",
  "messageId": "3EB0F1A9C2D4...",
  "timestamp": 1714567890
}
cURL
curl -X POST https://your-domain.com/api/send \
  -H "Cookie: st=TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"number":"919876543210","message":"Hello!"}'
Python
import requests

r = requests.post(
  "https://your-domain.com/api/send",
  cookies={"st": "TOKEN"},
  json={"number": "919876543210",
       "message": "Hello!"}
)
Body Parameters
FieldTypeRequiredDescription
numberstringRequiredRecipient phone with country code, no + (e.g. 919876543210)
messagestringRequiredText content to send. Supports WhatsApp formatting (*bold*, _italic_)
POST /api/send-media Send an image or document ›

Send an image, PDF, or other file to a WhatsApp number. Upload the file as a multipart/form-data request. An optional caption is shown below the media.

200 OK500 WA Offline401 Unauthorized
cURL (multipart)
curl -X POST https://your-domain.com/api/send-media \
  -H "Cookie: st=TOKEN" \
  -F "number=919876543210" \
  -F "caption=Here is your invoice" \
  -F "file=@/path/to/invoice.pdf"
Response
{
  "status": "sent",
  "messageId": "3EB0A9F7..."
}
Form Fields
FieldTypeRequiredDescription
numberstringRequiredRecipient phone with country code, no +
filefileRequiredThe file to send (image, PDF, audio, etc.)
captionstringOptionalText caption displayed below the media
GET /api/messages List recent conversation messages ›

Returns a list of recent messages across all conversations, proxied from the Go WhatsApp server. Use the jid query parameter to filter by a specific contact.

200 OK401 Unauthorized
cURL
# All conversations
curl https://your-domain.com/api/messages \
  -H "Cookie: st=TOKEN"

# Filter by contact
curl "https://your-domain.com/api/messages?jid=919876543210@s.whatsapp.net" \
  -H "Cookie: st=TOKEN"
Response
[
  {
    "id": "3EB0F1...",
    "from": "919876543210@s.whatsapp.net",
    "message": "Hello, is this available?",
    "timestamp": 1714567890,
    "fromMe": false
  }
]
GET /api/events Real-time message stream via Server-Sent Events ›

Opens a persistent SSE connection that streams new incoming WhatsApp messages in real time. Use this to update your chat UI without polling. The connection stays open until the client disconnects.

SSE: Use the browser's native EventSource API — no WebSocket library needed.
JavaScript (EventSource)
const es = new EventSource('/api/events');

es.addEventListener('message', e => {
  const msg = JSON.parse(e.data);
  console.log(msg.from, msg.message);
});

es.addEventListener('status', e => {
  // connection / disconnection events
  console.log(JSON.parse(e.data));
});
Event Types
EventDescription
messageNew incoming WhatsApp message with from, message, timestamp fields
statusConnection status change (connected / disconnected)

Bulk Campaigns

Send a message to a large list of contacts. Each message is sent with a configurable delay (default 20 seconds) between sends to reduce the risk of being flagged by WhatsApp.

Anti-ban delay: Each message waits a random 20–60 seconds before the next send. The variation in timing reduces WhatsApp spam detection. Keep daily sends under 300 messages and only message opt-in contacts.
POST /api/campaigns Create a new bulk campaign ›

Creates a campaign with a contact list and a message template. The campaign starts in draft state — call /start to begin sending.

200 OK401 Unauthorized
Request Body
{
  "name": "Diwali Offer 2025",
  "message": "Hi {name}! Get 20% off this Diwali. Use code DIWALI20.",
  "contacts": "Ravi,919876543210\nPriya,918765432109\n917654321098"
}

// Delay is automatic: random 20–60s per message
Response
{
  "id": 7,
  "name": "Diwali Offer 2025",
  "status": "draft",
  "total": 3
}
Body Parameters
FieldTypeRequiredDescription
namestringRequiredCampaign display name
messagestringRequiredMessage text. Use {name} to personalise per contact
contactsstringRequiredOne contact per line. Format: Name,PhoneNumber or just PhoneNumber
delayautoFixedRandom 20–60 seconds between each message. Not configurable — keeps the account safe.
GET /api/campaigns List all campaigns ›

Returns all campaigns for the current user, ordered newest first. Each item includes the current status and delivery counts.

200 OK401 Unauthorized
Response
[
  {
    "id": 7,
    "name": "Diwali Offer 2025",
    "status": "running",
    "total": 150,
    "sent": 42,
    "failed": 1,
    "created_at": "2025-04-16T10:00:00"
  }
]
Campaign Status Values
StatusMeaning
draftCreated but not started. No messages sent yet.
runningActively sending messages in the background.
pausedManually paused. Can be resumed by calling /start again.
doneAll contacts have been processed.
POST /api/campaigns/{id}/start Start or resume a campaign ›

Starts sending messages for a draft or paused campaign. Sending runs in the background — the endpoint returns immediately. Already-sent contacts are skipped on resume.

200 OK400 Already Running401 Unauthorized
cURL
curl -X POST https://your-domain.com/api/campaigns/7/start \
  -H "Cookie: st=TOKEN"
Response
{
  "status": "running"
}
POST /api/campaigns/{id}/pause Pause a running campaign ›

Signals the background worker to stop after the current message finishes sending. The campaign moves to paused state and can be resumed at any time with /start.

200 OK401 Unauthorized
cURL
curl -X POST https://your-domain.com/api/campaigns/7/pause \
  -H "Cookie: st=TOKEN"
Response
{
  "status": "paused"
}
GET /api/campaigns/{id}/stats Live delivery statistics ›

Returns real-time delivery progress for a campaign. Poll this every few seconds while the campaign is running to update a progress bar in your UI.

200 OK401 Unauthorized
Response
{
  "id": 7,
  "name": "Diwali Offer 2025",
  "status": "running",
  "total": 150,
  "sent": 42,
  "failed": 1,
  "pending": 107,
  "percent": 28
}
DELETE /api/campaigns/{id} Delete a campaign and all its contact records ›

Permanently deletes a campaign and all associated contact delivery records. Running campaigns are stopped before deletion. This action cannot be undone.

200 OK401 Unauthorized
Response
{
  "status": "deleted"
}

Incoming Webhook

The Go WhatsApp server calls this endpoint whenever a new message arrives. Register /webhook/incoming as the callback URL in your Go server config.

POST /webhook/incoming Called by the Go server on each incoming message ›

This endpoint is invoked by the Go WhatsApp bridge when a message is received. It returns 200 OK immediately to prevent Go server timeouts, then processes the message asynchronously in a background thread.

Note: Group messages (@g.us) and media-only messages (no text) are silently ignored.
Payload sent by Go server
{
  "from": "919876543210@s.whatsapp.net",
  "message": "Hi, is this available?",
  "timestamp": 1714567890
}
Response (always immediate)
{
  "ok": true
}

// Processing happens async
// in a background thread
Ignored Message Conditions
ConditionReason
Sender ends with @g.usGroup messages are not processed
Empty message textMedia-only messages are skipped
Message starts with [System or status messages are skipped
