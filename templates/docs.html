Authentication

Whatfy uses session-based authentication. Every request must include your session token via the Authorization header or the st cookie set after login.

WhatsApp Endpoints

POST /api/wa/send Send a WhatsApp text message ›

Send a text message to any WhatsApp number. The number must include country code without the + prefix (e.g. 919876543210 for India).

200 OK 401 Unauthorized
Request Body
{
  "number": "919876543210",
  "message": "Your order #42 is confirmed!"
}
Response
{
  "status": "sent",
  "messageId": "3EB0F1A9...",
  "timestamp": 1714567890
}
cURL
curl -X POST \
  https://your-domain.com/api/wa/send \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"number":"919876543210","message":"Hello!"}'
Python
import requests

r = requests.post(
  "https://your-domain.com/api/wa/send",
  headers={"Authorization":"Bearer TOKEN"},
  json={"number":"919876543210",
       "message":"Hello!"}
)
Body Parameters
FieldTypeRequiredDescription
numberstringRequiredPhone number with country code, no + (e.g. 919876543210)
messagestringRequiredText content of the message to send
GET /api/wa/status Check WhatsApp connection status ›

Returns the current WhatsApp connection state including whether the session is active and the linked phone number.

200 OK401 Unauthorized
cURL
curl https://your-domain.com/api/wa/status \
  -H "Authorization: Bearer TOKEN"
Response
{
  "connected": true,
  "loggedIn": true,
  "phone": "919876543210",
  "name": "My Business"
}
GET /api/wa/qr-image Get QR code as base64 PNG ›

Fetches the current WhatsApp QR code from the Go server, converts it to a PNG image, and returns it as a base64 data URL. Show this in an <img> tag for users to scan.

Response
{
  "qr_image": "data:image/png;base64,iVBORw0KGgoAAAANSUh..."
}

// Error (already connected or no QR)
{
  "error": "No QR available — may already be connected"
}
POST /api/wa/pair Pair with a phone number ›

Request a pairing code for a given phone number. The user enters this code in WhatsApp → Linked Devices → Link with phone number.

Request
{
  "number": "919876543210"
}
Response
{
  "code": "ABCD-1234"
}
GET /api/wa/logout Disconnect the WhatsApp session ›

Terminates the WhatsApp session, clears all session tokens, and disconnects from the Go server. All users must reconnect afterwards.

Response
{
  "status": "disconnected"
}

Webhooks

POST /webhook/incoming Receive incoming WhatsApp messages ›

This endpoint is called by the Go WhatsApp server when a message is received. The AI engine processes the message in a background task and sends a reply. Returns 200 OK immediately to prevent Go server timeouts. Messages from groups or without text are silently ignored.

Payload from Go Server
{
  "from": "919876543210@s.whatsapp.net",
  "message": "Price of rice?",
  "timestamp": 1714567890
}
Response (always immediate)
{
  "ok": true
}

// AI reply sent async to WhatsApp
// via background task (no blocking)
Ignored Messages
ConditionReason
Sender ends with @g.usGroup messages are skipped
Empty message textMedia-only messages skipped
Message starts with [System/status messages skipped

AI Agent Endpoints

Query any AI agent with a natural language message. The agent responds based on your live business data (inventory, patients, invoices, appointments).

POST /api/ai/shop Shop inventory AI assistant ›

Query the Shop AI with any inventory or order question. The AI has full context of your current stock levels, prices, thresholds, and units.

Request
{
  "message": "Which items are low in stock?"
}
Response
{
  "reply": "3 items below threshold:\n• Rice – 2kg left (min 5kg)\n• Sugar – 1kg left (min 3kg)\n• Dal – 500g left (min 2kg)"
}
POST /api/ai/invoice Invoice AI assistant ›

Query the Invoice AI for help creating, reviewing, or managing invoices. The AI has context of your invoice history and draft status.

Request
{
  "message": "How many unpaid invoices do I have?"
}
Response
{
  "reply": "You have 3 draft invoices totalling ₹12,400 pending."
}
POST /api/ai/health Health & patient AI assistant ›

Query the Health AI for patient management, medication reminders, or medical image analysis. Pass a base64 image for AI vision analysis.

Request (text)
{
  "message": "Which patients need medication today?"
}
Request (image)
{
  "message": "Analyze this prescription",
  "image": "data:image/jpeg;base64,..."
}
Parameters
FieldTypeRequiredDescription
messagestringRequiredNatural language query or instruction
imagestringOptionalBase64-encoded image for AI vision analysis (Pro plan only)
POST /api/ai/appointments Appointment scheduling AI ›

Query the Appointment AI for scheduling, reminders, and managing appointments. The AI has context of all upcoming appointments and their statuses.

Request
{
  "message": "Who has appointments tomorrow?"
}
Response
{
  "reply": "Tomorrow: Ravi (10:00 AM, Dr. Kumar), Priya (3:30 PM, Dr. Singh). Send reminders?"
}

Configuration

POST /api/agent/{type} Enable or configure an AI agent ›

Toggle an agent on/off and set its configuration. The {type} path parameter must be one of: shop, invoice, health, appointment.

Request
{
  "enabled": true,
  "wa_number": "919876543210",
  "notes": "Custom system instructions..."
}
Response
{
  "status": "saved"
}
Path Parameters
ValueAgent
shopShop & Inventory Agent
invoiceInvoice Agent
healthHealth & Patient Agent
appointmentAppointment Agent
GET /api/dashboard/stats Fetch live dashboard statistics ›

Returns real-time stats: WhatsApp status, low-stock count, draft invoices, patient total, today's appointments, and agent configuration.

cURL
curl https://your-domain.com/api/dashboard/stats \
  -H "Authorization: Bearer TOKEN"
Response
{
  "wa_phone": "919876543210",
  "g_low": 3,
  "inv_draft": 2,
  "p_total": 48,
  "appt_today": 5,
  "agents": {
    "shop": {"enabled": true}
  }
}
